# 分類匹配指南

## 原則

優先使用現有分類 + `--note` 補充細節。建立新分類是**最後手段**，只有在品項完全無法歸入任何現有分類時才使用。

## 現有預設分類

| 分類 | Icon | 適用 |
|------|------|------|
| 餐飲 | 🍽️ | expense |
| 交通 | 🚗 | expense |
| 日用品 | 🧴 | expense |
| 娛樂 | 🎮 | expense |
| 醫療 | 🏥 | expense |
| 教育 | 📚 | both |
| 居住 | 🏠 | expense |
| 服飾 | 👕 | expense |
| 薪資 | 💰 | income |
| 投資 | 📈 | both |
| 其他 | 📦 | both |

> 執行 `categories` 指令取得最新清單，上表僅為預設值。

## 匹配流程

### Step 1: 從現有分類中選擇最接近的大分類

所有食物飲料相關 → **餐飲**。所有移動方式相關 → **交通**。以此類推。

### Step 2: 用 `--note` 記錄細節

餐別（午餐/宵夜）、購買地點（全聯/好市多）、交通用途等放 `--note`，不要放在分類名稱裡。

### Step 3: 只有完全無法歸類時才建立新分類

例如「寵物飼料」— 現有 11 個分類都不適合 → 可新建「寵物」。

## 錯誤示範（禁止）

| 使用者輸入 | ✗ 錯誤做法 | ✓ 正確做法 |
|-----------|-----------|-----------|
| 午餐 80 | add-category "午餐" | category=餐飲, note=午餐 |
| 飲料 50 | add-category "飲食" | category=餐飲 |
| 晚餐 79 | add-category "晚餐" | category=餐飲, note=晚餐 |
| 加油 1200 | add-category "汽油" | category=交通 |
| 買菜 450 | add-category "食材" | category=餐飲, note=買菜 |

## `--note` 常用值

- **餐飲**：早餐/午餐/晚餐/宵夜/下午茶/飲料；購買地點（全聯/好市多/外送）
- **交通**：目的地或用途（回家/上班/出差）
- **日用品**：購買通路（屈臣氏/全聯）或用途（清潔/文具）
- **娛樂**：訂閱/電影/遊戲 等子類型
- **通用**：任何有助於日後回顧的補充資訊

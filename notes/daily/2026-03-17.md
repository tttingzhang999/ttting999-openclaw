# 2026-03-17 — expense skill 首次上線除錯

## 背景

第一次在 Discord 透過 finance agent 記帳（午餐 80 恆逸便當），踩了一堆坑。

## 踩到的坑

### 1. Symlink 被 OpenClaw 靜默忽略

**症狀：** console 顯示 main 和 finance 都是 52 個 skill，expense 沒被計入。
agent 看起來有在用 expense.sh，但其實是透過 TOOLS.md 的文字描述找到路徑直接呼叫，不是正式載入 skill。

**原因：** OpenClaw skill discovery 做 realpath 安全檢查，symlink 解析後路徑在 workspace root 外面就會被忽略，不報錯。

**修法：** `ln -s` 改成 `cp -r`，實體複製 skill 到 workspace。

### 2. Agent 用錯參數（--item 而非 --desc）

**症狀：** finance agent 呼叫 `expense.sh add --item "恆逸的便當"`，但 CLI 只認 `--desc`。

**原因：** Skill 沒被正式載入（見上），agent 只看到 TOOLS.md 的簡要描述，自己猜了參數名。

**修法：**
- TOOLS.md 加了完整 CLI 參數簽名，標明「不可自創參數」
- SKILL.md Step 2 明確標註 `--desc`（不是 --item）

### 3. Agent 跳過查分類步驟

**症狀：** 直接用 `--category "午餐"` 呼叫 add，但「午餐」分類不存在。

**修法：** SKILL.md Step 3 加 ⚠️ 警告：「必須在 add 之前執行此步驟 — 不可跳過」

### 4. expense.sh add-category 的 jq 報錯

**症狀：** `jq: invalid JSON text passed to --argjson`

**原因：** `psql -tAc` 的 RETURNING 查詢會多輸出 command tag（如 `INSERT 0 1`），
導致 `$result` 是多行字串（`"28\nINSERT 0 1"`），傳給 `--argjson` 就炸了。

**修法：**
- 加了 `trim()` 函數去除前後空白
- `psql_cmd()` 加 `head -n1` 只取第一行資料
- `cmd_add` 裡的 `new_id`、`expense_total`、`income_total` 也加 trim

### 5. Discord exec approval 每次都要手動批准

**症狀：** 每次 agent 要執行 expense.sh 都卡在「Exec approval is required」。

**修法：**
```bash
openclaw approvals allowlist add --agent finance "*/skills/expense/scripts/*"
```
